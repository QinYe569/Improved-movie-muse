## 🎯 12 张数据库表的实际应用场景

### 1️⃣ **user 用户表**
**实际应用场景：**
- ✅ 用户注册（register.html）
- ✅ 用户登录（login.html）
- ✅ 个人中心信息展示（profile.html）
- ✅ VIP 会员系统（vip-system.js）
- ✅ 观看时长统计（VIP 升级依据）
- ✅ 用户状态管理（正常/禁用）

**后端接口需求：**
- POST /api/user/register - 用户注册
- POST /api/user/login - 用户登录
- GET /api/user/profile - 获取用户信息
- PUT /api/user/profile - 更新用户信息
- PUT /api/user/password - 修改密码
- PUT /api/user/vip - 更新 VIP 状态

**字段使用说明：**
| 字段 | 用途 |
|------|------|
| username | 登录账号、显示用户名 |
| password | 登录验证（需加密） |
| email | 注册邮箱、找回密码 |
| avatar | 用户头像 |
| is_vip | VIP 权限判断 |
| vip_expire_time | VIP 有效期验证 |
| status | 账号封禁处理 |

## 🔄 数据流向图

```
用户注册/登录 → user 表
    ↓
浏览电影 → movie 表 + category 表 + actor 表
    ↓
搜索电影 → search_history 表 + movie 表
    ↓
观看电影 → view_history 表 + movie 表（view_count+1）
    ↓
发表评论 → comment 表
    ↓
收藏电影 → favorite 表
    ↓
VIP 升级 → user 表（is_vip=1）
    ↓
管理员管理 → admin 表 + 各业务表
```

## 🎯 VIP 系统数据库实现方案

### 当前实现（前端 localStorage）
```javascript
// 每秒累加观看时长
localStorage.setItem('userWatchTime', newTime);
// 判断是否达到 VIP
if (newTime >= 360000) { // 100 小时
    localStorage.setItem('isVip', 'true');
}
```

### 后端实现方案
```sql
-- 1. 在 view_history 表中记录每次观看时长
INSERT INTO view_history (user_id, movie_id, view_progress, ...)
VALUES (1, 10, 3600, ...); -- 观看了 1 小时

-- 2. 计算累计观看时长
SELECT SUM(view_progress) as total_seconds
FROM view_history
WHERE user_id = 1;

-- 3. 判断是否升级 VIP
UPDATE user
SET is_vip = 1,
    vip_expire_time = DATE_ADD(NOW(), INTERVAL 365 DAY)
WHERE id = 1
  AND (SELECT SUM(view_progress) FROM view_history WHERE user_id = 1) >= 360000;
```
