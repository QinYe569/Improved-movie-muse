# Movie-Muse 数据库设计说明（12 张表版本）

## 📚 项目简介
Movie-Muse 是一个电影推荐与观看网站，适合大二 Web 开发课程项目。

**技术栈：**
- 前端：HTML + CSS + JavaScript
- 后端：Servlet + JSP
- 数据库：MySQL
- ORM 框架：MyBatis

---

### 5. movie_actor 电影演员关联表 ⭐ 新增
**用途：** 电影与演员的多对多关系

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INT | 主键，自增 | 1 |

| actor_id | INT | 演员 ID（外键） | 1 |
| role_name | VARCHAR(100) | 饰演角色 | 库珀 |

**主键：** id  
**外键：** 
- movie_id → movie(id)
- actor_id → actor(id)  
**唯一约束：** (movie_id, actor_id)

**示例数据：**
```
id | movie_id | actor_id | role_name
1  |    1     |    1     | 库珀
2  |    1     |    2     | 布兰德博士
```

**说明：** 这是数据库设计的重点，体现多对多关系

## 💡 常用 SQL 查询示例

### 1. 用户登录
```sql
SELECT * FROM user 
WHERE username = 'zhangsan' 
  AND password = MD5('123456')
  AND status = 1;
```

### 2. 获取电影列表（分页）
```sql
SELECT * FROM movie 
WHERE status = 1 
ORDER BY create_time DESC 
LIMIT 0, 20;
```

### 3. 获取某分类下的电影
```sql
SELECT m.* FROM movie m
INNER JOIN movie_category mc ON m.id = mc.movie_id
WHERE mc.category_id = 1 AND m.status = 1;
```

### 4. 获取电影的演员表
```sql
SELECT a.name, a.gender, ma.role_name
FROM actor a
INNER JOIN movie_actor ma ON a.id = ma.actor_id
WHERE ma.movie_id = 1
ORDER BY ma.id;
```

### 5. 获取电影详情（包含分类和演员）
```sql
-- 电影基本信息
SELECT * FROM movie WHERE id = 1;

-- 电影分类
SELECT c.name FROM category c
INNER JOIN movie_category mc ON c.id = mc.category_id
WHERE mc.movie_id = 1;

-- 电影演员
SELECT a.name FROM actor a
INNER JOIN movie_actor ma ON a.id = ma.actor_id
WHERE ma.movie_id = 1;
```

### 6. 获取用户的评论
```sql
SELECT c.content, c.create_time, m.title as movie_title
FROM comment c
INNER JOIN movie m ON c.movie_id = m.id
WHERE c.user_id = 1 AND c.status = 1
ORDER BY c.create_time DESC;
```

### 7. 统计用户观看次数
```sql
SELECT movie_id, COUNT(*) as watch_count
FROM view_history
WHERE user_id = 1
GROUP BY movie_id;
```

## 🔧 MyBatis 映射示例

### UserMapper.xml
```xml
<mapper namespace="org.example.mapper.UserMapper">
    
    <!-- 用户登录 -->
    <select id="login" parameterType="map" resultType="User">
        SELECT * FROM user 
        WHERE username = #{username} 
          AND password = MD5(#{password})
          AND status = 1
    </select>
    
    <!-- 用户注册 -->
    <insert id="register" parameterType="User" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO user (username, password, email, avatar, gender, create_time)
        VALUES (#{username}, MD5(#{password}), #{email}, #{avatar}, #{gender}, NOW())
    </insert>
    
</mapper>
```

```xml
<mapper namespace="org.example.mapper.MovieMapper">
```

## ✅ 设计亮点

1. **符合第三范式** - 演员表独立，避免数据冗余
2. **关系清晰** - 两个多对多关系表（movie_actor, movie_category）
3. **难度适中** - 适合大二学生，涵盖核心数据库知识
4. **易于扩展** - 预留了扩展字段
5. **教学价值高** - 体现数据库规范化设计思想

## 🎯 表拆分说明

### 拆分前（10 张表）
- movie 表包含 actors 字段（逗号分隔）

### 拆分后（12 张表）
- **actor 表** - 独立的演员信息表
- **movie_actor 表** - 电影与演员的关联表

### 拆分的优点
1. ✅ **避免数据冗余** - 演员信息不重复存储
2. ✅ **便于查询** - 可以查询演员的所有作品
3. ✅ **符合范式** - 符合数据库第三范式要求
4. ✅ **易于维护** - 修改演员信息只需改一处

## 📊 知识点总结

| 知识点 | 应用表 | 说明 |
|--------|--------|------|
| 主键 | 所有表 | PRIMARY KEY AUTO_INCREMENT |

| 唯一约束 | user, admin, category | UNIQUE |
| 索引 | 所有表 | INDEX idx_xxx |
| 默认值 | 所有表 | DEFAULT |

| 多对多关系 | movie_actor, movie_category | 数据库设计重点 |
| 第三范式 | actor 表 | 数据规范化 |
